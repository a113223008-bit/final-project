# 專案文件

## 一、專案簡介
本專案為一個以 Java 語言開發的「命令列餐廳訂餐系統」，
使用者可透過文字選單瀏覽菜單、搜尋菜色、加入購物車並進行結帳，
系統會自動產生訂單明細與總金額。

本系統主要目的在於練習 Java 基礎語法、
物件導向設計（OOP）、集合（Collection）及基本流程控制。

---

## 二、開發環境
- 程式語言：Java
- Java 版本：JDK 17（或以上）
- 開發工具：IntelliJ IDEA
- 執行方式：命令列（Console Application）

---

## 三、系統架構與類別設計
本系統主要由下列核心類別組成：

### 1. Main 類別
- 系統進入點，負責：
  - 顯示操作選單
  - 接收使用者輸入
  - 控制整體流程
- 使用 `Scanner` 讀取使用者指令
- 透過 `while(true)` 形成互動式選單

### 2. Restaurant 類別
- 負責餐廳層級的管理功能：
  - 菜單管理
  - 菜色搜尋
  - 訂單提交
- 提供以下主要功能：
  - 新增菜色至菜單
  - 依關鍵字搜尋菜色
  - 依 ID 查詢菜色
  - 接收並管理訂單

### 3. Dish 類別
- 代表單一道菜品
- 包含以下屬性：
  - dishId（菜品編號）
  - name（菜名）
  - price（價格）
  - isAvailable（是否可販售）
- 提供價格與販售狀態的存取方法

### 4. Order 類別
- 代表一筆訂單
- 主要功能：
  - 儲存多道菜品（List<Dish>）
  - 計算訂單總金額
  - 管理訂單編號與狀態

---

## 四、系統功能說明

### （一）初始化菜單
系統啟動時，會預先新增 10 道菜品至菜單中，
包含主食、飲料及小吃，方便使用者直接進行操作測試。

---

### （二）瀏覽與搜尋菜單
- 使用者可輸入關鍵字搜尋菜色
- 若直接按 Enter，則顯示全部菜單
- 系統會列出：
  - 菜品編號
  - 菜名
  - 價格
  - 是否已售罄

---

### （三）加入購物車
- 使用者輸入菜品 ID
- 系統檢查：
  - 菜品是否存在
  - 是否可販售
- 符合條件者可加入購物車清單

---

### （四）結帳與列印明細
- 若購物車為空，系統會提示無法結帳
- 成功結帳時：
  - 建立訂單物件
  - 計算總金額
  - 列印完整訂餐明細
- 結帳完成後會自動清空購物車

---

### （五）系統結束
- 使用者選擇退出選項後，系統即結束執行

---
